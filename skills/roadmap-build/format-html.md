# Format: HTML Roadmap View

**Audience:** All — single shareable artifact combining every perspective  
**Purpose:** One self-contained HTML file combining all four roadmap views with a live Gantt chart. Open in any browser, no tools required.  
**Filename:** `roadmap-view.html`

> This file defines the **combined HTML view** (option 6) and the **shared CSS** used by all individual HTML files. Every format file references this CSS block for its own `roadmap-*.html` output.

---

## When this format is generated

Only when the user selects option 6. Individual HTML files (`roadmap-now-next-later.html`, etc.) are generated by their respective format files regardless of this option.

**If the four markdown files already exist** → generate only `roadmap-view.html` (HTML-only mode).
**If any markdown file is missing** → generate all four markdown + individual HTMLs first, then `roadmap-view.html`.

---

## Content rules

- All story cycle values, week ranges, and totals MUST come from the shared estimate table computed in Step 7 — identical to the markdown formats.
- Apply the same Gantt construction rules from `format-gantt.md` (same IDs, same dates, same `crit` tags).
- Do not add content that does not appear in the other formats — the HTML is a presentation layer, not a new analysis.
- The file must open correctly with a plain double-click in any modern browser. Test that the Mermaid CDN script tag is valid.

### Changelog rules

- Omit the Changelog section and nav link if this is v1 and no changelog entries exist.
- Read changelog data from the `roadmap-changelog.md` file in the output directory (written by the versioning step in SKILL.md Step 9).
- List versions most recent first.
- Mark a changelog entry `breaking` if the reason indicates a commitment change, client-requested scope reduction, or timeline slip.
