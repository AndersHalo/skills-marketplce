# Format: Theme-Based

**Audience:** Stakeholders and client
**Purpose:** Communicate strategic direction grouped by lines of value — without technical details or exact dates.
**Filename:** `roadmap-theme-based.md` + `roadmap-theme-based.html`

---

## Required Sections

### Vision
1-2 sentences from the PRD on what problem the next phase solves.

### Company Goal
The business objective guiding the roadmap. One clear sentence connecting product work to business outcome.

### Timeline
ASCII bar chart showing which themes run in which weeks. Derive week ranges from the shared estimate table. Do NOT use quarter labels.

```
               Week 1-2         Week 3-5         Week 6-8         Week 9-11
               ─────────────────────────────────────────────────────────────
Theme 1        ████████
Secure Access

Theme 2        ████████████████████████████████
Delivery Intel

Theme 3                 ████████████████████████
Contract Confidence

Theme 4                                  ████████████████████████████████
Platform Maturity
```

Use actual theme names and week ranges from the shared estimate table.

### Per Theme — repeat for each theme (3–5 themes total)

**Theme naming rule:** Name themes after the customer problem being solved, not the feature being built.
- ✅ "Contribution Intelligence"
- ❌ "Add Jira Integration"

Structure for each theme:

1. **Theme name** as a heading
2. **Week range** — when this theme runs (e.g., `Weeks 1–3`)
3. *One sentence in italics* framing the customer problem this theme addresses
4. **Problem it solves** — 2-3 sentences explaining the pain without this theme
5. **Milestones** — 2-3 named milestones with week range and effort:
   ```
   **Milestone 1 — Name** *(Weeks {n}–{n})* · ~{n} story cycles
   ```
6. **Initiatives** — list with PRD references

   **If `any_sprint_coverage = true`**, add a status summary line after each initiative:
   ```
   - Initiative Name (FR-01) — In Progress · 60%
   ```

   **If `any_sprint_coverage = false`**, list initiatives without status:
   ```
   - Initiative Name (FR-01)
   ```

7. `**Theme effort:** ~{n} story cycles` after initiatives
8. **Success metric** — one measurable outcome that proves this theme delivered value

### Dependencies Between Themes
Prose paragraph explaining which themes depend on others and why.

### Effort Summary
Totals block per the estimation model spec.

## HTML Output

Generate `roadmap-theme-based.html` as a self-contained file. Use the full CSS from `format-html.md`. Structure:

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>{Product Name} — Themes</title>
  <!-- full CSS block from format-html.md -->
</head>
<body>

  <div class="hero">
    <h1>{Product Name}</h1>
    <p class="subtitle">Theme-Based Roadmap · {today's date} <span class="version-chip">v{version}</span></p>
    <div class="stats">
      <div class="stat"><span class="label">Themes</span><span class="value">{n}</span><span class="sub">total</span></div>
      <div class="stat"><span class="label">Story Cycles</span><span class="value">{n}</span><span class="sub">{n} parallel-optimized</span></div>
      <div class="stat"><span class="label">Calendar Time</span><span class="value">~{n} weeks</span><span class="sub">agentic pipeline</span></div>
      <div class="stat"><span class="label">Critical Path</span><span class="value">{n} cycles</span><span class="sub">{initiative A} → … → {initiative Z}</span></div>
      <!-- if any_sprint_coverage = true: -->
      <div class="stat"><span class="label">Sprint Status</span><span class="value">{n_complete} done</span><span class="sub">{n_in_progress} in progress · {n_planned} planned</span></div>
    </div>
  </div>

  <!-- Timeline bar -->
  <section>
    <p class="section-label">Visual Timeline</p>
    <h2>Themes at a Glance</h2>
    <div class="gantt-wrap"><pre>{ASCII timeline block}</pre></div>
  </section>

  <!-- one theme-card per theme -->
  <section id="themes">
    <p class="section-label">Stakeholders</p>
    <h2>Strategic Themes</h2>
    <p class="desc">{Company Goal}</p>

    <div class="theme-card ec{n}">
      <h3>{Theme Name}</h3>
      <span class="weeks-badge">Weeks {A}–{B}</span>
      <p class="problem">{Customer problem sentence}</p>
      <div class="milestone"><strong>{Milestone 1}</strong> · Weeks {n}–{n} · ~{n} cycles</div>
      <!-- if any_sprint_coverage: show initiative status inline -->
      <!-- if not any_sprint_coverage: show initiative list only -->
      <span class="effort-line"><strong>Theme effort:</strong> ~{n} story cycles</span>
    </div>
    <!-- repeat per theme -->
  </section>

  <!-- Effort Summary -->
  <section id="summary">
    <p class="section-label">Estimates</p>
    <h2>Effort Summary</h2>
    <!-- summary-grid cards per theme + totals card -->
    <!-- critical-path block -->
  </section>

  <footer>
    Generated by roadmap-build · {today's date} · v{version} · Sources: {documents used}
  </footer>

</body>
</html>
```

Apply `class="ec{n}"` on each `.theme-card` for the epic group color. Omit status details when `any_sprint_coverage = false`.
