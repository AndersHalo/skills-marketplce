# Format: Goal-Oriented (OKR)

**Audience:** Leadership and board
**Purpose:** Connect every initiative to a measurable business objective. Answer "why are we building this?" before "what are we building?"
**Filename:** `roadmap-goal-oriented.md` + `roadmap-goal-oriented.html`

---

## Required Sections

### Vision
1-2 sentences from the PRD on what problem the next phase solves.

### Per Objective — repeat for each objective (2–4 objectives total)

**Objective writing rules:**
- Must be ambitious and inspiring — not a task list item
- Must connect to a business outcome, not a feature delivery
- ✅ "Make visible the hidden cost of capacity that appears allocated but isn't contributing"
- ❌ "Build the Gap Calculation Engine"

Structure for each objective:

1. `## Objective {n}` heading with the objective statement
2. **Why it is ambitious** — 2-3 sentences on what makes this hard or meaningful
3. **Owner** — team or role responsible
4. **Timeframe** — week range derived from the shared estimate table (e.g., `Weeks 1–3`). Do NOT use quarter labels.
5. **Key Results table:**
   ```
   | Key Result | Target | How it is measured |
   ```
   - 3–5 KRs per objective
   - Each KR must be specific, quantifiable, and time-bound
   - "How it is measured" must name a concrete data source or observable signal

6. **Initiatives table:**

   **If `any_sprint_coverage = true`:**
   ```
   | Initiative | Timeframe | PRD Ref | Story Cycles | Confidence | Status | Completion % |
   ```

   **If `any_sprint_coverage = false`:**
   ```
   | Initiative | Timeframe | PRD Ref | Story Cycles | Confidence |
   ```

### Summary Table
One row per objective:
```
| Objective | Owner | Timeframe | Story Cycles | Critical KR |
```

### Effort Summary
Totals block per the estimation model spec.
